Posterita Unified Platform
Figma UX Brief

Product Goal
- Design one unified operational platform with one Android shell app for internal users and one web control console for admins and supervisors.
- The MVP focuses on Retail POS, Loyalty at POS, Close Till Reconciliation, Inventory Basic, Staff Ops, and Supervisor workflows.
- The platform is role-based and device-based. Users should only see modules enabled for their role and device.

Primary Users
- Cashier
- Store Staff
- Supervisor
- Admin or Owner

Core Product Principles
- Fast, touch-first, low-friction UI
- Role-based visibility
- Device-aware behavior
- Offline-friendly patterns
- Clear sync and lock states
- Safety-first for sensitive actions
- Minimal training required

Design Scope
- Android shell app
- Android POS workflows
- Android Staff Ops workflows
- Android Supervisor workflows
- Android Inventory Basic workflows
- Web control console

Do Not Design Now
- Full restaurant suite
- Marketing campaign builder
- Support inbox/chat platform
- Staff map tracking
- Advanced HR/payroll rules
- Deliveroo, ABSA, SalesIQ, Facebook, Instagram integrations
- Internal real-time chat

States to Design
Each major screen should include:
- loading
- empty
- success
- error
- offline
- syncing
- permission denied
- revoked or locked
- approval pending

Prototype Flows
1. Login -> device enrollment -> home
2. POS sale -> loyalty lookup -> payment -> receipt
3. POS close till -> discrepancy -> evidence upload -> submit
4. Staff leave request -> supervisor approval
5. Inventory count -> submit
6. Device revoked from web -> app locked state

Responsive Rules
- Android is phone-first and tablet-aware
- POS screens should also support tablet checkout comfortably
- Web console is desktop-first and tablet-secondary
- No need to fully optimize admin console for mobile in MVP
